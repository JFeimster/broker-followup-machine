Workflows

Seven playbooks that cover the core follow-up motion.

Each workflow should map to a clear stage, a review point, and a practical next action so the operator never has to guess what happens next.

New lead

Set the first response path, capture context, and define the next touch.

Missing docs

Keep the request sequence organized and easy to track.

Stalled deal

Re-engage quiet records with a sequence that does not feel random.

Declined reactivation

Use careful re-entry language and stop for review before any sensitive note goes out.

Funded renewals

Plan the post-close touchpoints and maintain long-tail contact discipline.

Referral nurture

Keep referral partners warm with steady, relevant follow-up.

Dead lead revival

Resurface old records with a clean path back into the pipeline.

Workflow structure

What each playbook should include

  • Trigger condition
  • Message objective
  • Channel order
  • Review checkpoint
  • Next-step rule

Operator view

How to keep it manageable

  • One workflow, one outcome
  • One owner, one next action

  • One clear history for the record
