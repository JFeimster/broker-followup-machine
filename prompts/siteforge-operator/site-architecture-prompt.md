# SiteForge Operator Prompt

## Purpose

Use this prompt with **SiteForge Operator** to create the website architecture, landing page flow, and conversion structure for **Broker Follow-Up Machine GPT**.

## Prompt

Create the website architecture for **Broker Follow-Up Machine GPT**.

## Goal

Convert loan brokers, business funding brokers, funding agency owners, and broker teams into users of the custom GPT.

## Product Positioning

Broker Follow-Up Machine GPT should be positioned as a broker follow-up operating system that creates:

- Sequences
- CRM stages
- Task ownership
- Automation rules
- Handoffs
- Stop conditions
- Human review points

## Required Output

Include:

- Sitemap
- Homepage section order
- Conversion flow
- Primary CTA
- Secondary CTA
- Lead magnet section
- FAQ
- SEO title and meta description
- Trust/safety section
- Workflow use case sections
- Recommended page copy structure

## Core Promise

Turn a messy broker pipeline into an automated follow-up machine that never forgets a borrower, referral partner, or funding opportunity.

## Audience

- Loan brokers
- Business funding brokers
- Funding agency owners
- Independent funding agents
- Referral partner managers
- Broker teams
- Processors
- Broker VAs

## Conversion Requirement

The site should make busy brokers understand the practical value quickly:

- Every lead needs an owner.
- Every deal needs a next action.
- Every sequence needs a stop condition.
- Every sensitive message should have a review checkpoint.

## Final Instruction

Design the site architecture around revenue protection, operational clarity, and broker-safe follow-up. Avoid generic AI landing page structure.
