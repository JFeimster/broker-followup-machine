# FLASH-UI Neo-Brutalist Landing Page Prompt

## Purpose

Use this prompt with **FLASH-UI** to generate a high-conversion, visually aggressive UI concept and implementation-ready front-end structure for **Broker Follow-Up Machine GPT**.

## Main Prompt

Design and generate a high-conversion, visually explosive web app UI for my product:

**Broker Follow-Up Machine GPT**

This is a custom GPT for loan brokers, business funding brokers, funding agency owners, independent funding agents, referral partner managers, broker teams, processors, and VAs.

## Core Promise

Turn a messy broker pipeline into an automated follow-up machine that never forgets a borrower, referral partner, or funding opportunity.

## UI Style

The UI style should feel like:

- Neo-brutalist fintech
- Cyberpunk automation dashboard
- Futuristic revenue ops control center
- Bold, sticky, high-contrast SaaS
- Modern, edgy, conversion-focused
- Visually aggressive but still clean and usable

I want the UI to absolutely slap.

## Visual Direction

Combine these style influences.

### 1. Neo-Brutalism

- Hard edges
- Thick borders
- Sharp cards
- Oversized typography
- Punchy layout blocks
- Bold shadows
- Assertive spacing
- Tactile button styling

### 2. Fintech Futurist

- Premium dashboard feel
- Sleek data panels
- Workflow cards
- KPI badges
- Trust + precision
- Clean structured information hierarchy
- “Money moves through systems” feel

### 3. Cyberpunk Automation

- Glowing accents
- Electric color contrast
- Dark mode option or dark hero sections
- Neon signals for alerts, statuses, and workflow activity
- Subtle HUD-style overlays
- Automation map / pipeline motion feel
- “Mission control for broker follow-up”

## Mood

The product should feel:

- Fast
- Sharp
- Premium
- Operational
- Sticky
- Addictive
- Visually memorable
- Modern
- Like a serious tool, not a toy

It should not feel:

- Bland
- Corporate
- Generic SaaS template
- Soft pastel startup
- Boring CRM clone
- Crypto scammy
- Cluttered
- Cheesy AI art
- Impossible to read

## Core UI Goal

The page should make brokers instantly feel:

> “Damn. This solves the exact operational mess I deal with.”

And:

> “I need this in my stack.”

## Color System

Use vivid contrasting colors that pop.

Primary palette should include combinations like:

- Electric blue
- Neon green
- Hot magenta
- Safety orange
- Cyber yellow
- Deep black / charcoal
- Off-white / light gray for contrast sections

Suggested color direction:

- Background: matte black / deep charcoal / graphite
- Surface cards: black, dark gray, or stark white depending on section
- Accent 1: electric cyan
- Accent 2: neon lime
- Accent 3: hot pink / magenta
- Alert accent: orange or yellow
- Text: bright white on dark sections, deep near-black on light sections

Important:

Use color boldly, but preserve readability and hierarchy. Do not flood every section with all colors at once. Use strong contrast and intentional accent placement.

## Typography

Typography should be:

- Oversized
- Bold
- Assertive
- Compressed or geometric for headlines
- Crisp and clean for body text

### Headline Style

- Huge
- Powerful
- Punchy
- Almost poster-like
- Short line lengths
- High impact

### Body Text

- Readable
- Practical
- Slightly dense but clean
- Clear enough for conversion copy

## Layout Style

The layout should be dynamic and high-energy.

Use:

- Large headline blocks
- Broken grid sections
- Asymmetrical compositions
- Sticky nav
- Sticky CTA rail or sticky demo card
- Bold section dividers
- Stacked cards
- Layered panels
- “Dashboard meets landing page” structure

The UI must feel sticky in both senses:

1. Memorable and addictive
2. Actual sticky interface behavior

Include sticky elements like:

- Sticky top nav
- Sticky CTA button
- Sticky side panel on desktop
- Sticky floating “Use GPT” button
- Sticky progress indicator or section nav if useful

## Components to Include

Create a landing page / app interface that includes the following components.

## 1. Hero Section

Include:

- Massive headline
- Hard-hitting subheadline
- Primary CTA: Use Broker Follow-Up Machine GPT
- Secondary CTA: Download Sequence Pack

Hero visual should show:

- A bold pipeline dashboard
- Follow-up stages
- CRM-like cards
- Automation map
- Lead, docs, stalled deal, renewal, and referral partner statuses
- Glowing status chips
- Overdue alerts
- Ownership tags
- Action cards

## 2. Workflow Cards

Use large brutalist cards for:

- New Lead Follow-Up
- Missing Document Follow-Up
- Stalled Deal Revival
- Dead Lead Reactivation
- Declined Borrower Reactivation
- Funded Client Renewals
- Referral Partner Nurture
- CRM Cleanup & Automation

Each card should feel clickable, tactile, and visually punchy.

## 3. Pipeline Visualization

Create a futuristic pipeline strip or board:

- New Lead
- Contact Attempted
- Application Started
- Documents Requested
- Broker Review Needed
- Submitted
- Renewal Watch
- Referral Follow-Up

Each stage should look alive. Use badges, motion, glows, or progress states.

## 4. Automation / Sequence Section

Show:

- Trigger
- Condition
- Action
- Wait
- If/then
- Stop condition
- Human review

This should look like a visual automation builder, not plain text.

## 5. Before / After Section

### Before

Messy, forgotten, leaking pipeline.

### After

Owned, visible, automated, routed, reviewed.

Make this section dramatic and visual.

## 6. Safety / Human Review Section

Create visually strong cards showing:

- “Human review required”
- “Do not auto-send terms”
- “No approval promises”
- “No lender-specific claims”
- “Pause before sensitive communication”

This should feel sharp and trustworthy, not boring.

## 7. CTA Section

Big, loud, irresistible. Use high contrast and strong visual framing.

## Interaction / Motion Style

Use modern motion and micro-interactions:

- Hover lift
- Glow response
- Slide transitions
- Subtle pulse on active CTA
- Moving highlight borders
- Animated card focus
- Smooth sticky transitions
- Parallax or depth layers if appropriate

Do not overdo animation. Motion should support clarity and energy.

## Design Rules

- Make every section visually distinct
- Keep the page highly scannable
- Use strong visual hierarchy
- Prioritize conversion
- Make buttons obvious
- Use shadows, borders, and glows intentionally
- Keep spacing confident and sharp
- Make the UI look custom, not template-generated

## Content Tone

Use direct, practical, revenue-focused language. This is for brokers.

Avoid hypey fake marketing language.

Tone should be:

- Slightly blunt
- Clear
- Operational
- Persuasive
- Grounded
- Implementation-focused

## Key Product Messaging

Use messaging like:

- Stop losing broker deals to forgotten follow-up
- Your pipeline is not broken. Your follow-up system is.
- Every lead needs an owner
- Every deal needs a next action
- Every sequence needs a stop condition
- Chase missing docs without sounding pushy
- Revive stalled deals without sounding desperate
- Build follow-up systems that protect revenue
- Turn manual follow-up chaos into a machine

## UX Priorities

- Conversion-first
- Mobile responsive
- Fast loading
- Highly memorable
- Sticky CTA presence
- Premium visual identity
- Excellent contrast
- Strong readability
- Feels like a category-defining tool

## Output Requirement

Generate a polished, high-fidelity UI concept and implementation-ready front-end structure for this product.

The result should feel like a hybrid of:

- Premium SaaS landing page
- Futuristic CRM control center
- Neo-brutalist product showcase
- Cyberpunk automation dashboard

## Final Quality Bar

The UI should feel bold enough to stop a scroll, sharp enough to impress a founder, and structured enough to convert a broker.
