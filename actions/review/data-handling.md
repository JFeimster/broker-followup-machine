# Data Handling

## Purpose

Describe the data handling boundary for Actions planning docs.

## When to use

- A workflow touches borrower or partner data.
- A schema or payload example is being drafted.

## Rules

- Do not add real borrower financial data.
- Do not add SSNs, tax records, credit reports, or bank statements.
- Keep examples generic and reviewable.

## Next implementation step

Mirror these rules in any future examples or payload notes.
