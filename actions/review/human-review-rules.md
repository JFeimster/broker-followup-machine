# Human Review Rules

## Purpose

Define the mandatory review checks for any Action planning output.

## When to use

- Before a draft becomes a sendable message.
- Before a sync note is treated as final.

## Review points

- Borrower-facing claims
- Lender-specific statements
- Underwriting-related messages
- Decline-related messages
- Sensitive document handling

## Output

- Human review required note
- Safe-to-edit draft
- Next action for the operator

## Next implementation step

Add these review checkpoints to every action folder before any live integration is considered.
