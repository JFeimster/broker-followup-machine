# Renewal Reminder

## Purpose

Describe a renewal reminder event for post-close planning.

## When to use

- A funded record approaches a planned touchpoint.
- A reminder draft should be queued.

## Inputs

- Funded date
- Reminder window
- Record owner

## Outputs

- Reminder task
- Draft-only message
- Review note

## Safety boundary

- Human review required before borrower-facing send.
- Do not imply certainty or outcome language.

## Next implementation step

Tie the event to a reviewable calendar or task path.
