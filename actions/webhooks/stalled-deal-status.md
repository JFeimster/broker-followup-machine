# Stalled Deal Status

## Purpose

Describe a stalled-deal event for re-engagement planning.

## When to use

- A record has been quiet beyond the expected window.
- A revival workflow should start.

## Inputs

- Record status
- Inactivity marker
- Owner

## Outputs

- Re-engagement task
- Draft-only message note
- Review checkpoint

## Safety boundary

- No borrower-facing claims.
- Human review required before send.

## Next implementation step

Use the event only to trigger planning, not delivery.
