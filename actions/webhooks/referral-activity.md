# Referral Activity

## Purpose

Describe a partner activity event for referral nurture planning.

## When to use

- A partner goes inactive.
- A partner check-in is due.

## Inputs

- Partner record
- Last activity date
- Follow-up cadence

## Outputs

- Nurture task
- Draft check-in
- Review note

## Safety boundary

- Keep the language partner-focused, not borrower-promising.
- Human review required before send.

## Next implementation step

Use the event to cue planning, not live messaging.
