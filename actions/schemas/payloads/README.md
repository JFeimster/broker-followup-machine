# Payload Planning

## Purpose

Describe payload fields and object shapes for future Actions.

## When to use

- You need to define request or event fields.
- You want the docs to stay narrow and reviewable.

## Inputs

- Workflow name
- Field names
- Data sensitivity

## Outputs

- Payload field list
- Required vs optional note
- Safety note

## Safety boundary

- Keep sensitive data out of examples.
- Human review required before any borrower-facing output.

## Next implementation step

Write the minimum fields first and avoid speculative extras.
