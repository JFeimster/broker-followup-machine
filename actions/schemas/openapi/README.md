# OpenAPI Planning

## Purpose

Use this folder for OpenAPI planning notes only.

## When to use

- You need to sketch request and response shape.
- You want a future schema without building a live integration.

## Inputs

- Endpoint intent
- Payload fields
- Review boundary

## Outputs

- Path note
- Method note
- Schema sketch

## Safety boundary

- Do not add executable API code.
- Do not store secrets.

## Next implementation step

Keep the schema minimal and align it with the narrowest workflow.
