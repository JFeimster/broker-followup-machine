# Schema Examples

## Purpose

Hold short examples of planned payloads or schema structures.

## When to use

- You need to show what the data should look like.
- You want to review the shape before implementation.

## Inputs

- Payload shape
- Workflow context
- Review requirement

## Outputs

- Example object
- Field notes
- Safety reminder

## Safety boundary

- No real borrower data.
- No secrets.

## Next implementation step

Use these examples to validate the planning docs, not to power a live connector.
