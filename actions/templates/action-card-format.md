# Action Card Format

Use this structure for each individual Action documented inside an Action catalog file.

## Action: [Action Name]

**Operation ID:**
[actionOperationId]

**Authentication Type:**
None

**Category:**
Lead Intake / Calculator / Template Generator / Webhook / Validation Helper

**Purpose:**
What this Action does.

**Best Used When:**
When the GPT should recommend or call this Action.

**Do Not Use When:**
Situations where this Action is unsafe or inappropriate.

**User Inputs Needed:**

- Input 1
- Input 2
- Input 3

**Suggested API Payload:**

- field_one
- field_two
- field_three

**Expected Output:**
What the Action should return.

**Human Review Required:**
Yes/No, with reason.

**Safe Usage Notes:**
Privacy, compliance, and workflow boundaries.

**Example User Request:**
Example phrase a user might say.

**Recommended GPT Behavior:**
What the GPT should do before and after the Action.
