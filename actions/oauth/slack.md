# Slack

## Purpose

Plan Slack-connected Actions for internal notifications and team handoffs.

## When to use

- The team needs a status notification.
- An internal handoff should stay within a user-connected workspace.

## Inputs

- Channel or message target
- Event name
- Review note

## Outputs

- Notification outline
- Message draft
- Operator review checkpoint

## Safety boundary

- Internal-only messaging guidance.
- Do not route borrower-facing claims through Slack.

## Next implementation step
