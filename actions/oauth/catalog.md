# OAuth Catalog

## Purpose

Document the OAuth path for user-connected tools and workspace actions.

## Use this when

- The action must run under a user account.
- The data lives in a connected workspace or CRM.
- A delegated connection is required.

## Inputs

- Connected app
- Scope list
- Human review requirement

## Outputs

- OAuth planning note
- Scope guidance
- Safety checkpoint

## Safety boundary

- Human review required before sensitive messages or claims.
- Do not add live auth flows here.

## Next implementation step

Define scopes before any service-specific planning.
