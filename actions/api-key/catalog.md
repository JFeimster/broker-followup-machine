# API Key Catalog

## Purpose

Document the API key Action path for simple service-to-service requests.

## Use this when

- The workflow needs a direct API call.
- The operator or team owns the key handling.
- OAuth is unnecessary.

## Inputs

- Endpoint name
- Request payload shape
- Operator review point

## Outputs

- Key-handled Action note
- Request outline
- Review checklist

## Safety boundary

- Human review required before sensitive sends.
- Never store or document real secrets here.

## Next implementation step

Describe the request surface first, then stop until the review path is approved.
