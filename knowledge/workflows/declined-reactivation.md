# Declined Reactivation

## Purpose

Revisit declined files with careful, review-aware language.

## When to use

- A prior file was declined and the team wants to re-open the conversation later

## Key rules

- Human review required before any decline-related message
- Keep language careful and respectful
- Do not promise a different result

## Supports GPT output consistency

- Reinforces a safe reactivation pattern for declined files
