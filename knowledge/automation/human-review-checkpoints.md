# Human Review Checkpoints

## Purpose

Define where a human must review before sending or changing status.

## When to use

- A workflow involves borrower-facing language
- A workflow touches decline-related, underwriting-related, or lender-specific content
- Sensitive documents are involved

## Key rules

- Human review required before send
- Human review required before status changes that affect borrower communication
- Human review required before sensitive-document handling

## Supports GPT output consistency

- Keeps automation advice aligned with manual review boundaries
