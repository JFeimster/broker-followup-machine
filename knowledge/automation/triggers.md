# Triggers

## Purpose

Describe the events that should start a workflow or task.

## When to use

- A workflow needs a trigger condition
- The GPT needs to explain when a follow-up sequence should start

## Key rules

- Tie triggers to observable events
- Do not automate sensitive decisions
- Human review required when the trigger affects borrower-facing output

## Supports GPT output consistency

- Keeps trigger logic aligned to actual workflow events
