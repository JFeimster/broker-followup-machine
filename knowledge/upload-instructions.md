# Upload Instructions

Use this folder as the upload source for GPT knowledge.

## Before upload

- Review the safety files first
- Remove any content that implies approval, qualification, eligibility, rates, terms, or guarantees
- Keep examples short and consistent
- Keep sensitive document handling manual

## Upload order

1. `knowledge/core/`
2. `knowledge/safety/`
3. `knowledge/workflows/`
4. `knowledge/messaging/`
5. `knowledge/automation/`
6. `knowledge/examples/`

## File rules

- Use plain Markdown
- Keep each file focused on one job
- Use headers and bullet lists instead of long prose
- Prefer operator language such as "human review required" and "draft only"
- Do not recommend sending sensitive documents through insecure channels

## Outcome

The upload set should help the GPT produce consistent broker follow-up drafts, workflow guidance, and review-aware output without drifting into risky claims.
